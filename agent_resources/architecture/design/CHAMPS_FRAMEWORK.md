## Overview

CHAMPS is an AI-powered tagging system that generates clear, actionable success criteria for micro-steps in task management. Adapted from Randy Sprick's classroom management framework, CHAMPS provides structure and clarity specifically optimized for ADHD brains.

### What Makes CHAMPS Different?

Traditional task management systems focus on WHAT you need to do. CHAMPS focuses on:
- **HOW** to do it (Activity)
- **WHERE** you need to be (Movement)
- **WHAT ENERGY** it requires (Participation)
- **WHEN** you'll need help (Help)
- **WHO** you'll interact with (Conversation)
- **HOW YOU'LL KNOW** you're done (Success)

### Core Principle

> **Clear expectations reduce cognitive load and anxiety, enabling ADHD brains to start and complete tasks more effectively.**

## Why CHAMPS?

### The ADHD Challenge

ADHD brains struggle with:
1. **Task Initiation**: "I don't know where to start"
2. **Working Memory**: "What was I supposed to be doing?"
3. **Sustained Attention**: "How much longer will this take?"
4. **Completion Uncertainty**: "Am I done? Did I do it right?"
5. **Context Switching**: "This task requires different energy than I have"

### The CHAMPS Solution

By providing explicit answers to six key questions, CHAMPS removes ambiguity and reduces the cognitive load required to:
- **Start** a task (clear expectations)
- **Maintain** focus (known duration/energy)
- **Complete** successfully (clear success criteria)
- **Choose** the right task for the moment (context matching)

### Research Foundation

The classroom CHAMPS framework has demonstrated:
- **Improved concentration** in students with ADHD
- **Increased task completion** rates
- **Reduced anxiety** around task expectations
- **Better time-on-task** performance

**Source**: Randomized controlled trial with 102 teachers and 1,450 middle school students (Sprick et al.)

## How It Works

### 1. Tag Generation Pipeline

```
User captures task: "Buy groceries for the week"
        â†“
Task decomposed into micro-steps by DecomposerAgent
        â†“
For each micro-step:
    1. Extract: description, estimated_minutes, leaf_type
    2. Call CHAMPSTagService.generate_tags()
    3. LLM analyzes step context
    4. Returns CHAMPS tags for all 6 dimensions
    5. Tags stored in micro_steps.tags (JSON array)
        â†“
Tags displayed in UI, used for filtering/analytics
```

### 2. LLM Prompt Strategy

The CHAMPS service uses a specialized prompt that:
- Explains each dimension with examples
- Provides emoji-based tag vocabulary
- Considers step duration for Participation tags
- Considers leaf_type (DIGITAL vs HUMAN) for Activity tags
- Generates 2-4 tags per dimension
- Includes reasoning and confidence score

**Example LLM Output**:
```json
{
  "conversation": ["ğŸ’¬ Communication"],
  "help": ["âœ… Verify", "ğŸ“‹ Organize"],
  "activity": ["ğŸ›’ Purchase", "ğŸš— Travel"],

  "reasoning": "Grocery shopping requires travel, physical movement, communication with checkout, sustained focus (15 min), and clear completion (items acquired)",
  "confidence": 0.92

```

### 3. Fallback System

If LLM generation fails:
- Keyword-based tag generation
- Duration-based Participation tags
- Safe default tags per dimension
- Confidence score reduced to 0.6

## Implementation Architecture

### Database Schema

```sql
-- Existing table (already implemented)
CREATE TABLE micro_steps (
    step_id TEXT PRIMARY KEY,
    parent_task_id TEXT NOT NULL,
    step_number INTEGER NOT NULL,
    description TEXT NOT NULL,
    estimated_minutes INTEGER NOT NULL,
    icon TEXT,
    short_label TEXT,
    leaf_type TEXT DEFAULT 'HUMAN',
    delegation_mode TEXT DEFAULT 'DO',
    status TEXT DEFAULT 'TODO',
    tags TEXT,  -- JSON array of CHAMPS tags
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP,
    FOREIGN KEY (parent_task_id) REFERENCES tasks(task_id)
);

CREATE INDEX idx_micro_steps_tags ON micro_steps(tags);
```

### Service Architecture

**CHAMPSTagService** ([src/services/champs_tag_service.py](src/services/champs_tag_service.py)):
- LLM-powered tag generation (OpenAI + Anthropic)
- Structured output with Pydantic models
- Fallback keyword matching
- Confidence scoring

**MicroStepService** ([src/services/micro_step_service.py](src/services/micro_step_service.py)):
- Integrates CHAMPSTagService
- Auto-generates tags on step creation
- Provides tag retrieval APIs

**DecomposerAgent** ([src/agents/decomposer_agent.py](src/agents/decomposer_agent.py)):
- Calls CHAMPS generation during decomposition
- Ensures all micro-steps have tags

### Data Models

```python
class CHAMPSTags(BaseModel):
    conversation: List[str] = Field(default_factory=list)
    help: List[str] = Field(default_factory=list)
    activity: List[str] = Field(default_factory=list)
    movement: List[str] = Field(default_factory=list)
    participation: List[str] = Field(default_factory=list)
    success: List[str] = Field(default_factory=list)

    def get_all_tags(self) -> List[str]:
        """Flatten all tags into single list"""
        return (self.conversation + self.help + self.activity +
                self.movement + self.participation + self.success)

class CHAMPSTagResult(BaseModel):
    tags: CHAMPSTags
    reasoning: str
    confidence: float = Field(ge=0.0, le=1.0)
```
